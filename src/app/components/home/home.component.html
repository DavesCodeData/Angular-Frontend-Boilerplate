Let's start with what is Angular. Specifically, Angular CLI. This is a piece of software that uses the Type Script langauge to code a website. Angular CLI comes with lots of neat modules to make coding easier once you know how to use them in your Type Script.

Here you can see my organizational pattern. I have my website organized into components, services, models and run them in the app files.

Getting started now lets look at the modules

On lines 5-12 I have created all of these modules to do specific tasks. We will go over each of them. The modules above line 5 are imports from the .angular file package specifically the angular-webpack package.

The imported Angular modules such as HttpClientModule are incredibly useful and necessary in creating a full stack development. This is a crucail module to import because, it is what allows us to connect this application to my Node.js server which is another program I created for handling the restAPI's get, post, put and delete commands. But we'll get to that later.

Next we will look at routing

Routing works by creating paths to follow so that the application can run through the approprate components and from there run my CSS, HTML, and Type Script files in the correct order to display content for you to see.

You may have noticed that the HomeComponent does not have the name of any component in its path and has been left blank! This is entirly on purpose. Leaving the path to your home page blank is a convention of setting the websites display to the HTML and CSS of the home component for whenever there is no specified path. But there is a little more to this in that the app needs specification as to what the default view in app.component.html will display.

So lets look at that next!

The apps html file

This might not look like much but really this is the first html file of many to be read by the computer. Here we set up the design of the website in that the header component will be displayed on top and the footer component will be displayed on the bottom.

So you might be wondering "what is router-outlets?" And the answer is that these outlets are my components. Each component has it's own html file that uses CSS for position and gets it's materials either from Angular materials or through assests I created and placed in the assets folder such as the images you are seeing on the left. The specifics as to how these files are read will come from each components unique component.ts file and the configuration of the component will come from each components unique component.spec.ts file.

From here lets take a look at the first selection of this HTML file which is the header.

The header component

That is the name that we can be selected by in the app.component.html file and now run my headers HTML file and CSS file for this component.

With this in mind it is finally time to start seeing the app-routing.module.ts file in action by taking a look at the headers HTML file.

Positioning with CSS

Here you might recognize that many of the names with the periods in front of them are the same names as the classes in my HTML file. That's because, this is a CSS file and I am using the classes of my HTML file for positioning and color.

Now that you know how I position the content I want to display from my HTML file you have seen in large part how an Angular component works. There is a little bit more to this but we'll get to that later.
